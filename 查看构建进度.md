## 🔍 如何查看构建状态

### 1. 进入 Actions 页面
访问上面的链接，你会看到构建任务列表

### 2. 查看构建状态
每个构建任务都有一个状态图标：
- 🟡 **黄色圆点** - 正在构建中
- ✅ **绿色对勾** - 构建成功
- ❌ **红色叉号** - 构建失败
- ⚪ **灰色圆圈** - 尚未开始

### 3. 点击构建详情
点击最新的构建任务，可以看到：
- 构建日志
- 每个步骤的执行情况
- 预计剩余时间

## ⏱️ 构建时间

### 首次构建
**约 8-10 分钟**

原因：
- 下载 Gradle
- 下载依赖库
- 编译代码
- 生成 APK

### 后续构建
**约 5-7 分钟**

原因：
- Gradle 和依赖已缓存
- 只需要编译代码

## 📱 构建成功后如何下载 APK

### 第1步：进入构建详情
1. 在 Actions 页面，点击最新的构建任务
2. 选择带有 **绿色 ✓** 的构建

### 第2步：找到 Artifacts
滚动到页面**最底部**，找到 **"Artifacts"** 区域

你会看到两个文件：
| 文件名 | 说明 | 推荐 |
|--------|------|------|
| `yueshengshi-debug` | Debug 版本 | 测试用 |
| `yueshengshi-release` | Release 版本 | ✅ 推荐 |

### 第3步：下载 APK
1. 点击文件右侧的 **下载图标** 📥
2. 下载的是 `.zip` 文件
3. 解压后得到 `.apk` 文件

---

## 📊 构建流程说明

GitHub Actions 会自动执行以下步骤：

```
1. ✅ 检出代码 (Checkout code)
2. ✅ 设置 JDK 11 (Set up JDK 11)
3. ✅ 下载 Gradle 和依赖 (Download dependencies)
4. ✅ 编译 Debug APK (Build Debug APK)
5. ✅ 编译 Release APK (Build Release APK)
6. ✅ 上传 APK (Upload APK)
```

你可以在构建日志中看到每个步骤的进度。

## ❓ 常见问题

### Q1: 构建一直卡住不动
**A:** 检查：
- 网络连接是否正常
- 是否还在正常下载依赖
- 可以尝试重新触发构建

### Q2: 构建失败怎么办
**A:** 查看构建日志：
1. 点击失败的构建任务
2. 查看错误信息
3. 常见原因：
   - 依赖下载失败 → 重新触发构建
   - 代码错误 → 修复代码后重新推送

### Q3: 找不到 Artifacts
**A:** 检查：
- 构建是否成功完成（绿色 ✓）
- 是否滚动到页面最底部
- Artifacts 是否有30天保留期限

### Q4: 如何重新构建
**A:** 两种方法：

**方法1：手动触发**
1. 进入 Actions 页面
2. 点击左侧的 "Build APK" 工作流
3. 点击右侧的 "Run workflow"
4. 选择分支，点击 "Run workflow"

**方法2：推送新代码**
修改代码后，重新推送：
```bash
git push
```

### Q5: 构建/下载速度慢怎么办
**A:** 这是正常的：
- GitHub Actions 在云服务器上构建
- APK 文件大小约 5-8 MB
- 网络速度会影响下载时间
