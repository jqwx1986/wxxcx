# 🚀 快速开始 - 上传代码并构建 APK

## 前提条件

### 1. 安装 Git（如果还没有）

**下载 Git：**
- 官网：https://git-scm.com/download/win
- 或点击这里直接下载：https://github.com/git-for-windows/git/releases/download/v2.44.0.windows.1/Git-2.44.0-64-bit.exe

**安装 Git：**
1. 运行下载的安装程序
2. 一路点击 "Next"（使用默认设置即可）
3. 安装完成后重启命令提示符

---

## 第一步：创建 Personal Access Token（必须！）

### 1.1 访问 Token 生成页面
点击以下链接（需要登录 GitHub）：
https://github.com/settings/tokens

### 1.2 生成新 Token
1. 点击 `Generate new token` → `Generate new token (classic)`

### 1.3 配置 Token
填写以下信息：

| 选项 | 值 |
|------|-----|
| Note | `Git Upload Token` |
| Expiration | `90 days` 或 `No expiration` |
| Select scopes | ✅ 勾选 `repo`（这会自动选中所有 repo 权限） |

### 1.4 生成并复制
1. 点击页面底部的 `Generate token` 按钮
2. ⚠️ **立即复制 Token！**（只显示一次）
3. 保存到记事本，后面要用

---

```
C:\Users\文兄\CodeBuddy\20260310170730
```

## 第三步：等待自动构建

### 3.1 访问 Actions 页面
点击以下链接：
https://github.com/jqwx1986/wxxcx/actions

或者手动访问：
1. 进入你的 GitHub 仓库
2. 点击顶部的 `Actions` 标签

### 3.2 查看构建状态
你会看到：
- 🟡 正在运行（黄色圆点）
- ✅ 构建成功（绿色对勾）
- ❌ 构建失败（红色叉号）

### 3.3 等待时间
- **首次构建**：8-10 分钟
- **后续构建**：5-7 分钟

你可以关闭页面，几分钟后回来查看。

---

## 第四步：下载 APK

### 4.1 进入构建详情
1. 在 Actions 页面，点击最新的构建任务（通常在最上面）
2. 点击带有绿色 ✓ 的构建

### 4.2 找到下载链接
滚动到页面最底部，找到 **"Artifacts"** 区域

你会看到两个文件：
- `yueshengshi-debug` - Debug 版本（测试用）
- `yueshengshi-release` - Release 版本（推荐）✅

### 4.3 下载并解压
1. 点击文件右侧的下载图标 📥
2. 下载的是 `.zip` 文件
3. 解压后得到 `.apk` 文件

---

## 第五步：安装到手机

### 5.1 传输 APK
选择一种方式：
- USB 数据线
- 微信/QQ 发送到手机
- 云盘（百度网盘、OneDrive）

### 5.2 允许安装未知来源
1. 手机打开 `设置`
2. 搜索"安装未知来源"
3. 找到文件管理器，设置为"允许"

### 5.3 安装
1. 在手机上找到 `.apk` 文件
2. 点击安装
3. 完成后打开应用

### 5.4 测试
点击"立即打开"按钮，应该会自动跳转到粤省事小程序。

## ❓ 常见问题

### Q1: 脚本提示"未检测到 Git"
**A:** 安装 Git 并重启命令提示符：
https://git-scm.com/download/win

### Q2: 推送时身份验证失败
**A:** 检查以下几点：
- 用户名是否是 `jqwx1986`
- 密码是否是 Token（不是 GitHub 密码）
- Token 是否有 `repo` 权限
- Token 是否已过期

### Q3: 找不到 Token 怎么办
**A:** Token 只在创建时显示一次。如果丢失了，需要重新生成：
https://github.com/settings/tokens

### Q4: 构建失败怎么办
**A:** 查看构建日志：
1. 进入 Actions 页面
2. 点击失败的构建任务
3. 查看错误信息

常见错误：
- Gradle 配置问题 → 重新触发构建
- 依赖下载失败 → 重新触发构建

### Q5: APK 安装失败
**A:** 检查：
- 手机系统版本（需要 Android 7.0+）
- 是否允许"未知来源"应用安装
- APK 文件是否完整

### Q6: 如何重新构建新版本
**A:** 修改代码后，重新运行上传脚本即可
